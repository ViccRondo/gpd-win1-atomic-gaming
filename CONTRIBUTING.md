# Contributing

1. Keep changes specific, reviewable, and free of credentials or game data.
2. Run `scripts/validate.sh` before opening a pull request.
3. Record real-hardware evidence for display, input, audio, suspend, and power
   changes. Process logs alone do not prove that a visible feature works.
4. Prefer an upstream fix in Gamescope, MangoHud, Decky, Mesa, or Bazzite when
   the change is not GPD Win 1-specific.
5. Do not describe untested hardware paths as supported.
